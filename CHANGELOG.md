# Universal LLM Prompt V2 - Changelog

## Что нового в V5

### 🎯 Версия V5 - Dual Model Architecture
**Ключевое улучшение:** Раздельная архитектура Vision + Text модели

### 🚀 Основные фичи V5:
- **Dual Mode (Vision + Text)** — два отдельных API вызова: сначала Vision модель анализирует изображение, затем Text модель генерирует улучшенный промпт
- **Single Mode (Multimodal Text)** — одна мультимодальная модель для текста и изображений
- **Раздельные статус-выходы** — `vision_status` и `text_status` для прозрачного мониторинга
- **Кэширование system prompt** — интеллектуальное кэширование, пропускает отправку system prompt если он не изменился
- **Smart endpoint construction** — умное построение endpoint URL, избегает двойного /v1/
- **Streaming support** — поддержка streaming ответов от API

## Что нового в V2

### 🚀 Async/Await - главное улучшение!
**Проблема V1:** UI зависает на 120 секунд во время API вызова
**Решение V2:** Async запросы - UI остается отзывчивым

```python
# V1 - блокирует UI
result = requests.post(...)  # UI мертв 120 сек

# V2 - не блокирует UI  
result = await aiohttp.post(...)  # UI работает!
```

### 🎯 ComfyNodeABC - современная типизация
**Улучшения:**
- Полная типизация всех параметров
- Автодополнение в IDE
- Лучшая валидация
- Стандартизированный интерфейс

```python
# V1
"prompt": ("STRING", {"multiline": True})

# V2
"prompt": (IO.STRING, {"multiline": True})
```

### 📊 Улучшенная обработка ошибок
- Более понятные сообщения об ошибках
- Детальное логирование
- Таймауты с правильной обработкой

### 🔧 Технические улучшения
- Type hints для всех функций
- Async context managers
- Правильная обработка event loop
- UTF-8 encoding по умолчанию

## Миграция с V1 на V2

### Шаг 1: Найти ноду
В ComfyUI найдите "Universal LLM Prompt V2 (Async)"

### Шаг 2: Заменить
1. Удалите старую ноду V1
2. Добавьте новую V2
3. Переподключите входы/выходы

### Шаг 3: Проверить
Все параметры идентичны - настройки можно скопировать

## Когда использовать V1 vs V2

### Используйте V2 (рекомендуется):
- ✅ Всегда, если нет проблем
- ✅ Для длинных API запросов
- ✅ Когда нужна отзывчивость UI
- ✅ Для production использования

### Используйте V1 (только если):
- ⚠️ V2 не работает (сообщите о баге!)
- ⚠️ Нужна совместимость со старым кодом
- ⚠️ Проблемы с async в вашей среде

## Бесплатные провайдеры для тестирования

- Без API ключа
- Полный контроль
- Приватность
